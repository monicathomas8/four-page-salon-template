# Contributing Guidelines

## Branching
- Always create your branch directly from the issue
- Never create a branch manually

## Workflow
- All work must be done on your own branch
- Merge into **dev** first for testing
- Only merge into **main** when tested and approved

## Pull Requests
- Every merge requires a pull request
- At least 1 approval is needed before merging
- The person who wrote the code cannot approve their own pull request

## Issues
- All work must have an issue before a branch is created
- Issues should be clear and descriptive
- Assign yourself to the issue when you start working on it

## Commit Messages
- Keep commit messages short and clear
- Describe what you did e.g. `add hero section` or `fix navigation link`
